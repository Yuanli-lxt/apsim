# 齐河县多年化肥统计约束与 APSIM 施肥逻辑

## 1. 结论与使用边界

德州统计年鉴已提供齐河县 2015—2023 年连续的农用化肥实物量和折纯量，可用于约束县域氮素投入的数量级及年际变化。但该表是全县、全部农业用途的年度合计，不区分小麦、玉米和其他作物，也不提供施用日期。因此它不能直接替换 APSIM 的作物季施氮方案。

本研究采用三层逻辑：

1. **统计约束层**：由年鉴给出县域年度化肥实物量和折纯量；
2. **作物分配层**：显式假设有多少化肥氮分配给冬小麦—夏玉米，不把其他作物投入错误分给轮作；
3. **时期分配层**：把作物季总氮拆分为播种肥和追肥，这是管理情景，不由年度统计表反演。

原 `ordinary_farmer` 中小麦和玉米各 180 kg N/ha 的方案仅作为先前测试情景保留，以保证已完成基线可复现；它不是必须维持的目标、优化约束或县级统计观测值。本阶段不修改品种、作物生理、土壤或模型结构。

完整的实物氮、磷、钾、复合肥及折纯分项见 `qihe_multiyear_official_fertilizer_statistics.csv`。2018、2019和2023年分项之和与折纯合计相差 1 吨，是整数舍入差异，原样保留。

国家统计局的定义为：某种化肥折纯量等于实际施用量乘以包装标识的有效成分比例；氮肥折为 N，磷肥折为 P2O5，钾肥折为 K2O，复合肥按所含主要成分折算。官方解释见 <https://www.stats.gov.cn/hd/cjwtjd/202302/t20230207_1902270.html>。

例如100 kg尿素含氮46%，折纯氮为46 kg N；100 kg标称15-15-15的复合肥，折纯有效养分为45 kg，其中N、P2O5、K2O各15 kg。“折纯氮肥”列只代表氮肥类别的折纯量；“折纯复合肥”列是复合肥主要有效养分合计，并不等于其中的氮。年鉴没有报告齐河县复合肥配方构成，因此不能唯一拆成N、P2O5和K2O。

## 3. 县域 N–P2O5–K2O 质量守恒诊断

探索性估算使用：

```text
估算县域N = 折纯氮肥 + 折纯复合肥 × 复合肥N占比
估算县域P2O5 = 折纯磷肥 + 折纯复合肥 × 复合肥P2O5占比
估算县域K2O = 折纯钾肥 + 折纯复合肥 × 复合肥K2O占比
麦玉米播种面积占比 =（小麦播种面积 + 玉米播种面积）/ 农作物总播种面积
按面积分配的轮作养分 = 县域养分 × 麦玉米播种面积占比
```

复合肥构成设置三个敏感性情景：N:P2O5:K2O为1:1:1、4:3:3和2:1:1。年鉴不能判断哪个情景真实。

麦玉米分配比例也不是年鉴观测。现在把逐年的麦玉米播种面积占比作为可计算的中性参照；例如2020年为150,746.7/170,800=0.8826。该算法等价于假定所有作物单位播种面积施肥强度相同，只适合作为参照。0.70、0.80、0.90和1.00继续作为附加敏感性边界，并明确标记为固定比例假设。

180 kg N/ha只保留为结果表中的参考线，用于与旧测试情景比较。诊断不再反求“维持180所需比例”，也不据此筛选或优化年度施氮量。

## 4. 为什么当前产量模拟只施用 N

统计整理必须包括N、P2O5和K2O；当前APSIM产量响应却只能可靠分析N。原因是现用模板包含SoilN及初始硝态/铵态氮，但没有土壤Phosphorus剖面和SoilP模块，也没有用于本小麦—玉米模型的土壤K供给与K胁迫模块。

界面下拉框出现 `rock_p`、`banded_p`、`broadcast_p`，仅说明通用Fertiliser组件能发送P肥事件；要产生可信的P供应和作物P胁迫，还需要启用SoilP，并提供分层labile P、sorption等参数。当前模板不满足这些条件。该列表也没有K肥类型。因此不能仅选择下拉框中的P肥，就声称模型完成了P或K响应模拟。

现阶段处理方式为：N进入现有APSIM氮循环；P2O5、K2O保留在统计约束、年度趋势和管理合理性诊断中，不进入产量模型。以后如获得齐河土壤有效磷及必要的SoilP参数，可建立一个新的P限制情景并与冻结基线并行，不能直接改写当前基线。

## 5. 优化后的 APSIM 管理规则

新增的施肥预算验证在模型文件生成前执行：

- 每个作物必须显式声明 `total_n_kg_ha`；
- `sowing_n_kg_ha + topdress_n` 必须与总氮相等，否则停止生成模型；
- 追肥可用绝对量 `amount_kg_ha`，也可用 `fraction_of_total_n`，二者不能同时出现；
- 负值、非法日期、0—1范围外的比例均被拒绝；
- APSIM 输入单位始终为 kg N/ha，不是尿素产品质量；
- 县域统计约束不自动改写现有配置，也不按年份暗中调参。

当前播种/追肥比例保持不变：普通情景小麦为 108+72 kg N/ha，玉米为 60+120 kg N/ha。以后若选择某个有证据支持的年度总氮情景，可保持比例缩放，也可根据逐年管理资料替换时期分配；二者必须在 run metadata 中注明。

## 7. 下一步运行方案

多年 APSIM 原始运行应至少并行保留两类施肥情景：

1. **旧测试情景**：所有年份沿用既有180 kg N/ha，只用于复现和隔离气象年际响应，不要求后续方案维持该数值；
2. **统计约束敏感性**：按复合肥N–P2O5–K2O构成和轮作分配方法形成年度养分范围；其中只有N可进入当前APSIM，P2O5和K2O用于管理诊断；
3. **独立管理情景**：若获得逐作物施肥调查或测土配方资料，再据此定义小麦、玉米总氮及分次比例，不从县域总量强行反演唯一答案。

在获得作物专用化肥量、农户调查或逐年施肥制度前，不设置唯一“统计优化施肥量”。县级总量适合做质量守恒、异常检查和情景边界，不足以支持作物级最优施肥反演。
