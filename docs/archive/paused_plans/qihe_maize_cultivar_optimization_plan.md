# 齐河玉米品种参数优化与跨尺度验证计划

> 暂停说明：当前固定系数与管理不确定性研究阶段不优化品种参数、不修改作物生理参数。
> 本文仅作为历史规划保留，不是当前执行方案。

## 1. 背景与核心判断

当前齐河县普通农户空间基线在 2018—2020 年的玉米原始县域面积加权产量约为
10.68—12.53 t/ha，明显高于同期县级统计单产 7.26—7.57 t/ha。2020 年统计乘法校正系数
0.604069 能修正平均量级，但在独立的 2018、2019 年仍分别产生 -6.62% 和 -14.34% 的偏差，
因此该系数不能代替品种和作物过程参数校准。

现有玉米品种参数尚未完成有观测约束的系统优化。模型模板当前调用的玉米品种名称为
`P01_shandong_2025_v503_joint_iter60`，而玉米过程观测来自 2025 年 P02 点位。首先需要确认该名称
只是历史命名，还是实际引用了不对应的点位/品种参数。

同时必须区分两个不同问题：

1. **2025 P02 点位的玉米品种与过程参数是否正确**；
2. **齐河县统计平均产量为什么低于 APSIM 可实现产量**。

P02 的 2025 年实测收获产量约为 14.44 t/ha，属于较高产量水平，并不接近 2018—2020 年
7.26—7.57 t/ha 的县级统计均值。因此，即使 P02 品种参数优化成功，也不应预设它会自动消除县域
产量高估。县域差距还可能来自农户管理差异、非最优田块、灾害与病虫草害、播种面积口径、收获
损失，以及 APSIM 未表示的限制因素。

## 2. 总体目标

建立一套可追溯、分阶段、避免参数补偿的玉米优化流程：

1. 用 P02 的物候、LAI、生物量和产量观测校准 2025 年点位玉米品种参数；
2. 在不使用县级统计乘法系数的情况下，检查优化参数对 2018—2020 年县域模拟的影响；
3. 将“品种参数误差”和“县域管理/现实产量差距”分开量化；
4. 保留独立数据检验，避免同时用一批数据拟合又宣称验证；
5. 最终形成点位品种参数集、县域代表性方案、不确定性范围和完整复现记录。

## 3. 数据分工

| 数据 | 主要用途 | 是否用于拟合 | 说明 |
|---|---|---:|---|
| 2025 P02 五期 LAI | 冠层动态校准 | 是 | 当前日期为 7月16日至9月15日 |
| 2025 P02 五期叶、茎、总生物量 | 生物量形成与分配校准 | 是 | 应先统一鲜重/干重和单位口径 |
| 2025 P02 收获产量 14.44 t/ha | 籽粒形成校准 | 是 | 同一产量的 kg/亩和 kg/ha 两行只计一次 |
| 2025 P02 物候日期 | 热时间和光周期校准 | 待补充后使用 | 至少需要出苗、抽雄/吐丝、成熟日期 |
| P02 土壤、气象和管理记录 | 固定外部输入 | 否 | 播期、密度、施氮、灌溉必须尽量使用实测记录 |
| 2018—2020 齐河县统计单产 | 跨尺度迁移诊断 | 否 | 不直接用于 P02 品种参数拟合 |
| 新增年份或新点位过程观测 | 真正独立验证 | 否 | 优先增加另一个年份或另一个玉米点位 |

在只有一个点位、一个生长季的情况下，不能把全部品种参数同时自由优化。否则多个参数组合可能
得到相同产量，却对应错误的物候、LAI和生物量过程。

## 4. 实施阶段

### 阶段 0：冻结当前基线并核验品种身份

任务：

- 保存当前 `.apsim`、`Maize.xml`、气象、土壤、管理配置和模拟输出的副本或校验值；
- 记录 APSIM Classic 版本、可执行文件路径、运行时间和 Git 状态；
- 确认 `P01_shandong_2025_v503_joint_iter60` 的来源、真实品种名称及其是否对应 P02；
- 新建清晰的临时品种名，例如 `P02_maize_2025_calibration_v001`；
- 确保优化脚本不会永久覆盖 APSIM 安装目录中的全局 `Maize.xml`。

完成标准：基线能够重复运行，且优化前后的参数差异可以逐项追踪。

### 阶段 1：整理 P02 校准数据和输入

任务：

- 检查五期 LAI、叶生物量、茎生物量、总生物量的单位、干物质口径和重复样本；
- 把 kg/亩与 kg/ha 的收获产量识别为同一观测，防止目标函数重复加权；
- 补录播种、出苗、抽雄、吐丝、成熟和收获日期；
- 补录品种名称、种植密度、行距、施肥日期与用量、灌溉和前茬；
- 固定 P02 的气象、土壤初始水氮和管理输入，并建立输入质量检查表；
- 运行当前品种，生成逐日期观测—模拟基线表。

优先补充的是物候日期。没有物候观测时，热时间参数的可辨识性较弱，应缩小其范围或暂不优化。

### 阶段 2：建立候选参数清单与合理边界

不直接优化所有 XML 参数，而是按生理作用分组：

1. 物候参数：出苗至幼穗分化、开花、成熟等阶段热时间和光周期响应；
2. 冠层参数：叶片发生、潜在叶面积、叶片衰老和最大 LAI 相关参数；
3. 生物量参数：辐射利用、干物质分配和茎叶增长相关参数；
4. 籽粒参数：最大粒数、潜在粒重、灌浆速率和收获指数相关参数。

具体参数名必须以当前 APSIM Classic `Maize.xml` 的实际字段为准，不凭经验手工编造。参数上下界
优先来自 APSIM 默认品种范围、已有本地参数、文献和农艺常识；不建议一律使用固定百分比扩展。

### 阶段 3：敏感性筛选

先使用已有 Morris/Sobol 工作流筛选对下列输出敏感的参数：

- 开花日和成熟日；
- 五期 LAI；
- 叶、茎和总生物量；
- 籽粒产量、粒数和粒重。

第一轮使用 Morris 做低成本筛选；仅将敏感且有观测约束的参数进入 Sobol 或优化。最终自由参数
建议控制在 5—8 个，参数较多时分阶段优化，不进行一次性联合搜索。

### 阶段 4：分阶段多目标优化

优化顺序如下：

#### 4.1 先校准物候

在播期、管理和气象固定后，拟合出苗、抽雄/吐丝和成熟日期。物候没有基本对齐前，不进入产量
优化，因为错误的生育期可能通过其他参数补偿出看似正确的产量。

#### 4.2 再校准 LAI 与生物量

固定或窄化已确定的物候参数，使用五期 LAI、叶生物量、茎生物量和总生物量调整冠层形成与干物质
分配。需要同时检查峰值和时间轨迹，不能只比较季节最大值。

#### 4.3 最后校准籽粒形成和产量

在物候与生物量达到合理水平后，再优化粒数、粒重、灌浆和产量相关参数。产量不能获得压倒性的
权重，否则容易牺牲过程正确性。

目标函数建议对不同量纲先标准化：

```text
Score = w1 × 物候日期标准化误差
      + w2 × LAI 标准化误差
      + w3 × 生物量标准化误差
      + w4 × 产量标准化误差
      + 参数越界或生理不合理惩罚
```

初始可令四类权重接近，之后做权重敏感性检查。优化算法可复用项目已有的 SCE-UA/本地搜索能力，
但必须输出每次运行的参数、各分目标误差和综合分数，而不只保存一个总分。

### 阶段 5：点位验证与参数不确定性

由于目前 2025 P02 是唯一完整玉米季节，第一轮只能做弱验证：

- 将五个观测日期分为校准子集和留出子集，检查 LAI/生物量时间插值能力；
- 对最优参数附近保留多个近优解，而不是只报告唯一最优值；
- 通过重复起点或参数后验分布判断结果是否稳定；
- 新增另一年份或点位后，将其完整保留为独立验证集。

建议的阶段性评价阈值，而非无条件验收标准：

- 关键物候日期 MAE 不超过 5—7 天；
- LAI 的归一化 RMSE 不超过约 20%—25%；
- 总生物量归一化 RMSE 不超过约 25%—30%；
- P02 收获产量相对误差不超过 10%—15%；
- 不允许以明显错误的物候或生物量过程换取产量达标。

### 阶段 6：2018—2020 县域迁移试验

将优化后的 P02 品种参数直接替换到 5 km 县域基线，管理、土壤、气象和轮作掩膜保持不变，至少
比较四套结果：

| 方案 | 目的 |
|---|---|
| 当前品种、无统计校正 | 原始基线 |
| P02优化品种、无统计校正 | 判断品种优化本身的影响 |
| 当前品种、固定2020乘法系数 | 现有统计后校正基准 |
| P02优化品种、管理/产量差距模型 | 后续县域代表性方案 |

主要指标为逐年 Bias、MAE、RMSE 和 MAPE，并检查年际变化方向。只有两年独立统计验证时不重点
解释 R²。

若 P02 优化后仍接近 12—14 t/ha，而县均仍为 7—8 t/ha，应得出“品种优化不能解释县均差距”
的结论，继续构建普通农户管理分布或现实产量差距模型，而不是强行把品种参数调到县级均值。

### 阶段 7：县域代表性与现实产量差距

在品种过程参数合理后，再处理县域平均问题：

- 建立普通、较高产和低投入/受限农户的管理类型及面积权重；
- 纳入播期、密度、施氮、灌溉和初始水氮的空间或概率分布；
- 在有证据时加入病虫草害、渍害、倒伏、收获损失等 APSIM 未表示因素；
- 用多年统计数据约束县域分布，但保留年份做交叉验证；
- 分别报告 APSIM 可实现产量、管理加权产量和统计校正产量，避免概念混用。

## 6. 建议时间安排

| 周次 | 工作 | 里程碑 |
|---:|---|---|
| 第1周 | 冻结基线、核验品种、清洗P02数据、补管理和物候 | 获得可信的优化输入表 |
| 第2周 | 参数清单、边界设计、当前品种基线运行 | 明确可优化参数和原始误差 |
| 第3周 | Morris/Sobol敏感性筛选 | 将自由参数缩减到5—8个 |
| 第4周 | 物候、LAI和生物量分阶段优化 | 获得过程合理的候选参数集 |
| 第5周 | 籽粒产量优化、不确定性与留出日期检查 | 形成P02优化参数 v1 |
| 第6周 | 2018—2020县域迁移、方案对比和报告 | 判断品种优化能否改善县域预测 |

如果物候、播期或管理记录尚未获得，第1周之后仍可先做基线和敏感性分析，但不宜立即开展全参数
优化；缺失数据应作为显式不确定性保留。

## 7. 第一轮立即执行清单

1. 核实 P02 的真实玉米品种和模板中 `P01_shandong_2025_v503_joint_iter60` 的来源；
2. 整理 P02 播期、密度、施肥、灌溉和物候日期；
3. 从现有观测表生成去重、统一单位的玉米校准表；
4. 用当前参数重跑 2025 P02，输出逐日物候、LAI、生物量、粒数、粒重和产量；
5. 形成“观测是否足以约束各参数组”的审计表；
6. 再决定第一轮自由参数和范围，开始敏感性分析。

本计划的首要成功标准不是把县级统计误差立即压到零，而是先得到生理过程可信、来源清楚、能够
独立检验的玉米品种参数；随后再解释从高产点位到全县普通农户平均水平之间的产量差距。
